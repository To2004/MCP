# Synthetic PI Dataset

## Source
- **Paper:** Prompt Injection Detection and Mitigation via AI Multi-Agent NLP Frameworks — Gosmar et al.
- **Link:** https://arxiv.org/abs/2503.11517
- **Year:** 2025

## Format & Size
- **Total samples:** 500 engineered injection prompts (50 per category across 10 attack categories)
- **Format:** Categorized prompt injection examples with per-prompt and aggregate metrics; processed through a 3-agent pipeline (Generator, Sanitizer, Enforcer)
- **Availability:** arXiv paper — https://arxiv.org/abs/2503.11517 (check for supplementary dataset release; prompts generated by OpenAI o3-mini)

## Data Structure

| Field/Column | Data Type | Example Values | Usable As-Is? | Notes |
|---|---|---|---|---|
| Attack category | categorical | Direct Override, Authority Assertions, Hidden Commands, Role-Play, Logical Traps, Multi-Step, Conflicting Instructions, HTML/Markdown Embeds, Hybrid, Social Engineering | Yes | 10 categories — clean taxonomy for risk classification |
| Injection prompt | string | 50 examples per category | Yes | The actual attack prompts; useful for testing MCP tool input validation |
| Pipeline stage results | categorical/numeric | Generator output, Sanitizer output, Enforcer output | Partially | 3-stage processing results; need to extract per-stage metrics |
| ISR (Injection Success Rate) | numeric (0-1) | Per-category success rates | Yes | How often the injection bypasses defenses |
| POF (Prompt Override Frequency) | numeric (0-1) | Per-category override rates | Yes | How often the injection overrides the original prompt intent |
| PSR (Prompt Sanitization Rate) | numeric (0-1) | Per-category sanitization rates | Yes | How effectively the Sanitizer agent neutralizes injections |
| CCS (Context Coherence Score) | numeric (0-1) | Per-category coherence scores | Yes | Whether the output remains contextually coherent after injection attempt |
| TIVS (Total Injection Vulnerability Score) | numeric (composite) | Aggregate vulnerability score; 45.7% reduction through layered mitigation | Yes | Composite score combining ISR, POF, PSR, CCS — directly transferable to MCP risk scoring |

## Proposed Risk Dimensions

### Prompt Injection Risk (by Attack Category)
- **Feeding columns:** Attack category, ISR per category, POF per category
- **Proposed scale:** 1-10 where 1 = injection category has near-zero success rate, 10 = injection category consistently bypasses all defenses
- **Derivation:** For each of the 10 attack categories, compute a risk score from ISR and POF. For MCP tools, this creates a 10-element risk profile vector — a tool might be resilient to Direct Override (score 2) but vulnerable to Social Engineering (score 8). Map ISR directly: 0-10% success = score 1-2, 10-30% = score 3-4, 30-50% = score 5-6, 50-70% = score 7-8, 70-100% = score 9-10.

### Input Sanitization Effectiveness
- **Feeding columns:** PSR (Prompt Sanitization Rate) per category
- **Proposed scale:** 1-10 where 1 = sanitization catches all injections, 10 = sanitization is ineffective
- **Derivation:** Invert PSR: high sanitization rate = low risk score. For MCP, this measures how well a tool's input validation layer performs. Score = 10 * (1 - PSR). If the tool has no input sanitization, default to score 10. This dimension is unique because it measures the *defense* rather than the *attack*.

### Context Integrity Risk
- **Feeding columns:** CCS (Context Coherence Score) per category
- **Proposed scale:** 1-10 where 1 = output context is always preserved correctly, 10 = output context is frequently corrupted by injection attempts
- **Derivation:** Invert CCS: high coherence = low risk. For MCP tools, context integrity matters because a successful injection that corrupts the output context can cascade through downstream tool calls in a multi-step agent workflow. Score = 10 * (1 - CCS). Weight this higher for tools that feed their output into other tools (chain risk).

### Composite Injection Vulnerability (TIVS-Derived)
- **Feeding columns:** TIVS (Total Injection Vulnerability Score), all sub-scores (ISR, POF, PSR, CCS)
- **Proposed scale:** 1-10 where 1 = minimal injection vulnerability, 10 = critically vulnerable to prompt injection
- **Derivation:** TIVS is already a composite score combining the four sub-metrics. Normalize TIVS to a 1-10 scale using the paper's baseline and mitigated values as anchors. The paper reports 45.7% reduction in TIVS through layered mitigation — use the unmitigated TIVS as the "worst case" (score 10 anchor) and the fully-mitigated TIVS as the "best achievable" baseline. For MCP, this is the single-number prompt injection risk score when you do not need the per-category breakdown.

### Multi-Layer Defense Effectiveness
- **Feeding columns:** Per-stage results from the 3-agent pipeline (Generator, Sanitizer, Enforcer), delta between unmitigated and mitigated TIVS
- **Proposed scale:** 1-10 where 1 = all defense layers are effective and complementary, 10 = defense layers fail to reduce vulnerability
- **Derivation:** Compute the TIVS reduction achieved by each pipeline stage. For MCP, this maps to a defense-depth risk dimension: a tool with only one layer of input validation scores higher than a tool with multiple complementary layers. Use the 45.7% reduction benchmark as a reference point — tools achieving less reduction score proportionally higher.

## Data Quality Notes
- 500 prompts is a moderate dataset size, but the even split (50 per category) means each category has limited examples. Statistical estimates per category will have wide confidence intervals.
- Prompts were generated by OpenAI o3-mini, which introduces a potential bias — the attack prompts reflect one model's understanding of injection techniques, which may miss attack patterns that other models or human red-teamers would find.
- The 3-agent pipeline (Generator, Sanitizer, Enforcer) is a specific architecture; results may not transfer directly to other defense setups. The TIVS reduction of 45.7% is specific to this pipeline configuration.
- The 10 attack categories are a useful taxonomy, but real-world prompt injections often combine multiple categories. The "Hybrid" category partially addresses this, but only 50 examples may not capture the full space of multi-vector attacks.
- Availability of the raw dataset is unclear — the paper is on arXiv but the 500 prompts may need to be extracted from supplementary materials or reproduced using the described generation method.

## Usefulness Verdict
This is the most directly actionable benchmark for the prompt injection dimension of MCP risk scoring. The 10-category attack taxonomy provides a structured classification system that maps cleanly to risk sub-scores — instead of a single "is this prompt injection?" binary label, you get a profile of which *types* of injection a tool is vulnerable to. The four metrics (ISR, POF, PSR, CCS) and their composite (TIVS) are a ready-made scoring framework that can be adopted almost directly into the MCP risk model.

The TIVS composite scoring approach is especially transferable. It demonstrates how to combine multiple sub-metrics into a single vulnerability score while preserving the ability to drill down into components. For the MCP 1-10 risk model, this pattern — composite score built from named sub-scores — is exactly what is needed across all risk dimensions, not just prompt injection. The 45.7% reduction finding also provides a useful calibration point: it shows that layered defenses meaningfully reduce but do not eliminate injection risk, which supports the 1-10 graduated scoring approach over a binary safe/unsafe classification.
