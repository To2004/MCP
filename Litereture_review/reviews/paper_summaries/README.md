## Summary Format

Each summary follows a consistent structure:

1. **Title and metadata** — Hebrew title, authors, venue, arXiv ID
2. **Core problem** — What is broken and why, with metaphors
3. **Methodology** — Technical approach, frameworks, experimental setup
4. **Key contributions** — Main findings with tables and diagrams
5. **Technical details** — Formulas, architectures, algorithms, code blocks
6. **Results** — Quantitative findings with comparison tables
7. **Limitations** — Critical analysis of weaknesses and gaps
8. **Relevance to thesis** — How the paper connects to MCP-RSS dynamic risk scoring
